## 核心能力：一个「私人智囊团」🧠

### 能力一：决策多视角验证

重大决策前，让 3-5 个不同领域的大脑同时审视——这种待遇，在现实生活中只有 CEO 才配拥有（而且还得付天价咨询费）。

```
「我想从大厂裸辞做独立开发」

→ 段永平："这是'对的事情'吗？Stop Doing List 里有这条吗？"
→ Naval："这个职业给你哪种杠杆？如果答案是'用时间换钱'，走人。"
→ Grove："你的关键产出指标是什么？裸辞会让它上升还是下降？"
→ 芒格："反过来想——什么情况下这个决定会让你后悔？"
```

花 0 元，享受 4 个顶级大脑的会诊。这买卖不亏吧？

### 能力二：思维框架的可迁移学习

当你反复使用某个 Skill，你实际上在**内化**它的思维习惯：

- 用 Naval 的"杠杆阶梯"分析职业选择 10 次后，你会自动问"这个选择给我什么杠杆"
- 用苏格拉底的"澄清追问"审视方案 10 次后，你会本能地追问"你说的 X 到底是什么意思"

**Skill 是脚手架，最终目标是拆掉脚手架，让思维方式长进你自己的大脑。** 到时候你就可以卸载这个仓库了（开玩笑的，留着备用）。

### 能力三：跨领域创意激发

最有价值的洞察往往来自**领域交叉**：

- 投资领域的"安全边际" → 迁移到工程决策："这个架构的故障安全边际是多少？"
- 神经科学的"多巴胺系统" → 迁移到产品设计："用户的奖励反馈周期是什么？"
- 哲学的"苏格拉底反诘" → 迁移到团队沟通："我们的共识是建立在什么假设上的？"

**创新不是凭空创造，是把 A 领域的思维工具，用到 B 领域的问题上。** 这个仓库就是你的「思维工具箱」。

## 使用方式 🛠️

### 方式一：作为 Agent Skill 安装

如果你使用支持 SKILL.md 格式的 Agent 工具（如 Qoder、Claude Code 等）：

```bash
# 克隆仓库
git clone https://github.com/alchaincyf/people.skill.git

# 安装全部 Skill（或只安装你需要的）
cp people.skill/skills/*.md ~/.agents/skills/

# 在对话中触发
# "帮我用段永平的视角分析一下这个投资机会"
```

### 方式二：作为思维框架参考阅读

每个 Skill 的 `.md` 文件本身就是一份**该人物的深度思维框架文档**。即使不装在 Agent 里，直接阅读也能系统学习其心智模型和决策启发式。

## 方法论：不是复制人，是提炼框架 🎯

每个 Skill 的调研和提炼遵循以下标准：

### 调研深度

| 人物 | 一手来源数量 | 核心材料类型 |
|:---|:---|:---|
| 段永平 | 20+ | 雪球问答、斯坦福演讲、浙大见面会、方三文访谈 |
| 费曼 | 40+ | 著作、访谈、演讲、书信、批评文献 |
| Naval | 40+ | 著作、播客、Twitter、访谈、批评 |
| 苏格拉底 | 30+ | 柏拉图对话录、Richard Paul 提问框架、CBT 现代应用 |

### 提炼标准

每个 Skill 必须回答五个问题：

1. **他用什么心智模型看世界？**（镜片）—— 3-7 个核心模型
2. **他用什么决策启发式做判断？**（直觉规则）—— 5-10 条可执行规则
3. **他怎么表达？**（DNA）—— 句式、语气、修辞习惯
4. **他绝对不会做什么？**（反模式）—— 明确边界
5. **什么是这个 Skill 做不到的？**（诚实边界）—— 承认局限

**关键区分**：捕捉的是 **HOW they think**，不是 **WHAT they said**。

## 使用场景示例 💡

### 场景一：周报/月报的综合分析

```
用户：帮我分析本周的工作得失，准备下周计划

→ 自动激活多个 Skill：
   - Andy Grove 检查产出指标和杠杆率
   - 芒格做跨主题综合分析和逆向检验
   - 段永平评估方向对不对
   
→ 输出：一份多视角审视的周报框架
```

### 场景二：重大人生决策

```
用户：我在考虑要不要接受这个海外工作机会

→ 段永平："这是'对的事情'吗？十年后回头看会不会后悔？"
→ Naval："这个选择给你什么杠杆？是劳动力杠杆还是无需许可的杠杆？"
→ 苏格拉底："你说的'更好的机会'，具体指什么？"
→ Grove："你的关键产出指标会因为这个选择改变吗？"

→ 输出：一个经过多视角压力测试的决策框架
```

### 场景三：内容创作审阅

```
用户：帮我看看这篇技术文章的逻辑是否严密

→ Paul Graham："核心论点能用一句话说清吗？"
→ 苏格拉底："你说的'性能优化'，具体指标是什么？"
→ 芒格："有没有你忽略的反面证据？"

→ 输出：逻辑漏洞识别 + 改进建议
```

## 如何贡献 🤝

people.skill 欢迎所有领域的思维框架贡献。

### 贡献标准

一个合格的人物 Skill 必须满足：

1. **一手来源 > 20 个**：书籍、访谈、演讲、论文、个人博客等原始材料
2. **可运行的心智模型**：不是名言摘抄，是可操作的认知框架
3. **明确的触发词**：用户能在什么场景下激活这个 Skill？
4. **诚实边界**：明确标注这个 Skill 做不到什么
5. **遵循标准结构**：心智模型 → 决策启发式 → 表达 DNA → 反模式 → 诚实边界

### 目录结构

```
skills/{name}-perspective.md
```

每个 Skill 是一个独立的 `.md` 文件，包含完整的 YAML frontmatter + 正文。
